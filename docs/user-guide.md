## ログイン

ブラウザで http://localhost:5173 を開くとログイン画面が表示されます。

| 項目 | 値（ローカル開発環境） |
|------|----------------------|
| グループ ID | `dev-group` |
| パスワード | `devpass` |

「ログイン」ボタンを押すと JWT トークンが発行され、ブラウザの localStorage に保存されます。ブラウザを閉じても再度開いたときにログイン状態が維持されます。

---

## デバイス一覧

ログイン後、左サイドバーにグループ内のデバイスが一覧表示されます。

- 緑の点 — デバイスがオンライン（AWS IoT に接続中）
- グレーの点 — デバイスがオフライン

デバイス名をクリックすると、そのデバイスのコマンド実行パネルに切り替わります。

---

## コマンドの実行

デバイスを選択すると「コマンド実行」パネルが表示されます。

1. テキストエリアにシェルコマンドを入力します
2. 「実行」ボタンを押すか Enter で送信します

```
例:
  uname -a
  ip addr show
  cat /etc/hostname
  ls -la /tmp
```

コマンドはデバイス上のシェルで実行されます。タイムアウトはデフォルト 30 秒です。

---

## 実行結果の確認

コマンド送信後、パネル下部に結果カードが追加されます。ステータスは自動でポーリングされます。

| ステータス | 意味 |
|------------|------|
| pending | デバイスへの通知待ち |
| running | デバイスが実行中 |
| completed | 正常終了（exit code 0） |
| failed | 異常終了（exit code ≠ 0） |

結果カードには以下が表示されます：
- 実行したコマンド
- 標準出力（stdout）
- 標準エラー出力（stderr）
- 終了コード
- 実行時間（ミリ秒）

> デバイスがオフラインの場合、コマンドは `pending` のまま保留されます。デバイスが再接続すると実行されます（保留は最長 7 日間）。

---

## ログアウト

サイドバー下部の「ログアウト」ボタンを押すと localStorage のトークンが削除され、ログイン画面に戻ります。
